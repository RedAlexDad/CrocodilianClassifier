# Скрипты для убийства процессов на порту 20128

Эти скрипты автоматически отслеживают и убивают процессы, занимающие порт 20128 (обычно процессы Yandex Browser Network Service).

## Скрипты

### 1. `kill_port_20128.sh` (Bash)

Простой bash-скрипт для однократного убийства процессов.

**Использование:**

```bash
./scripts/kill_port_20128.sh
```

**Особенности:**

- Логирует действия в `/tmp/kill_port_20128.log`
- Использует `lsof` для обнаружения процессов
- Использует `kill -9` и `fuser` для гарантированного убийства
- Проверяет результат

### 2. `kill_port_20128.py` (Python 3)

Более продвинутый скрипт с поддержкой непрерывного мониторинга.

**Использование:**

```bash
# Однократный запуск
./scripts/kill_port_20128.py

# Непрерывный мониторинг с интервалом 10 секунд
./scripts/kill_port_20128.py --continuous --interval 10

# С другим уровнем логирования
./scripts/kill_port_20128.py --log-level DEBUG
```

**Особенности:**

- Поддержка аргументов командной строки
- Непрерывный мониторинг (опция `--continuous`)
- Настраиваемый интервал проверки
- Логирование в файл и консоль
- Обработка ошибок и таймаутов

## Установка

1. Убедитесь, что скрипты исполняемые:

   ```bash
   chmod +x scripts/kill_port_20128.sh
   chmod +x scripts/kill_port_20128.py
   ```

2. Установите зависимости (для Python-скрипта зависимости не требуются, только стандартная библиотека).

## Автоматизация

### Запуск по cron

Добавьте в crontab для периодического запуска каждые 5 минут:

```bash
*/5 * * * * /home/redalexdad/GitHub/CrocodilianClassifier/scripts/kill_port_20128.sh
```

Или для Python-скрипта:

```bash
*/5 * * * * cd /home/redalexdad/GitHub/CrocodilianClassifier && ./scripts/kill_port_20128.py
```

### Запуск как демон (непрерывный мониторинг)

```bash
nohup ./scripts/kill_port_20128.py --continuous --interval 30 > /dev/null 2>&1 &
```

## Проверка порта

Вручную проверить, занят ли порт:

```bash
lsof -ti:20128
# или
ss -tulpn | grep :20128
# или
netstat -tulpn | grep :20128
```

## Примечания

- Процессы Yandex Browser могут автоматически перезапускаться. Для постоянного решения рассмотрите:
  1. Блокировку порта в настройках браузера
  2. Изменение конфигурации сети браузера
  3. Использование другого браузера для разработки

- Скрипты требуют прав на убийство процессов (обычно достаточно пользовательских прав, если процессы принадлежат тому же пользователю).

- Логи находятся:
  - Bash-скрипт: `/tmp/kill_port_20128.log`
  - Python-скрипт: `/tmp/kill_port_20128_py.log`
